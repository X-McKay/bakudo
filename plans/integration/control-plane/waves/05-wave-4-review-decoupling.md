# Wave 4: Review Decoupling & Host Artifacts

**Goal:** Decouple execution from review. After the worker finishes, the host must discover the worktree, inspect it to generate diffs/patches, run the review logic against the *worktree state* (not just the exit code), and then merge or discard the sandbox based on `profile.mergeStrategy`.

**Non-Goals:** Do not persist the sandbox lifecycle state to disk yet. That happens in Wave 5. This wave is strictly about the runtime pipeline within a single process.

## Pre-reads & Vocabulary
- Review `2026-04-19-bakudo-abox-control-plane-spec.md` (Section: Observation O-4 and The Review Pipeline).
- **Host Artifacts:** Files generated by the host (not the guest) that describe the result of the attempt: `patch.diff`, `changed-files.json`, `merge-result.json`.
- **Decoupled Review:** The concept that `reviewAttemptResult` runs *after* the sandbox has been preserved, using the worktree snapshot as input.

## Dependencies
- **Requires:** Wave 3 (Orchestration & Adapter Split).
- **Blocks:** Wave 5 (Persistence & UI).

## Step-by-Step Implementation

### 1. Create `worktreeInspector.ts`

This module runs `git diff` inside the discovered worktree to see what the agent changed.

```typescript
// src/host/worktreeInspector.ts
import { execFile } from "node:child_process";
import { promisify } from "node:util";
import type { WorktreeSnapshot } from "./worktreeDiscovery.js";

const execFileAsync = promisify(execFile);

export type WorktreeInspection = {
  hasChanges: boolean;
  changedFiles: string[];
  patchDiff: string;
};

export const inspectWorktree = async (
  snapshot: WorktreeSnapshot,
): Promise<WorktreeInspection> => {
  try {
    // 1. Get changed files
    const { stdout: nameStatus } = await execFileAsync("git", ["diff", "HEAD", "--name-only"], {
      cwd: snapshot.path,
    });
    
    const changedFiles = nameStatus
      .split("\n")
      .map((f) => f.trim())
      .filter((f) => f.length > 0);
      
    // 2. Get full patch
    const { stdout: patchDiff } = await execFileAsync("git", ["diff", "HEAD"], {
      cwd: snapshot.path,
    });
    
    return {
      hasChanges: changedFiles.length > 0,
      changedFiles,
      patchDiff,
    };
  } catch (err) {
    return { hasChanges: false, changedFiles: [], patchDiff: "" };
  }
};
```

### 2. Create `hostArtifactGenerator.ts`

This module writes the inspection results to the session directory.

```typescript
// src/host/hostArtifactGenerator.ts
import { writeFile } from "node:fs/promises";
import { join } from "node:path";
import type { WorktreeInspection } from "./worktreeInspector.js";

export const writeHostArtifacts = async (
  sessionDir: string,
  attemptId: string,
  inspection: WorktreeInspection,
): Promise<string[]> => {
  const artifacts: string[] = [];
  const attemptDir = join(sessionDir, "attempts", attemptId);
  
  if (inspection.hasChanges) {
    const patchPath = join(attemptDir, "patch.diff");
    await writeFile(patchPath, inspection.patchDiff, "utf8");
    artifacts.push("patch.diff");
    
    const filesPath = join(attemptDir, "changed-files.json");
    await writeFile(filesPath, JSON.stringify(inspection.changedFiles, null, 2), "utf8");
    artifacts.push("changed-files.json");
  }
  
  return artifacts;
};
```

### 3. Update `reviewer.ts`

Change the review logic to look at the inspection results.

```typescript
// src/host/reviewer.ts
import type { WorktreeInspection } from "./worktreeInspector.js";
import type { AttemptExecutionResult } from "../attemptProtocol.js";

export const reviewAttemptResult = (
  baseResult: AttemptExecutionResult,
  inspection: WorktreeInspection | null,
  mergeStrategy: "auto" | "interactive" | "none",
): AttemptExecutionResult => {
  // If the worker crashed, fail immediately
  if (baseResult.exitCode !== 0 && baseResult.exitCode !== null) {
    return { ...baseResult, status: "failed", summary: "Worker exited with non-zero status." };
  }
  
  // If read-only, success is just finishing
  if (mergeStrategy === "none") {
    return { ...baseResult, status: "succeeded", summary: "Task completed." };
  }
  
  // If it's an implement task, it MUST have changes
  if (!inspection || !inspection.hasChanges) {
    return { ...baseResult, status: "failed", summary: "Task completed but no files were changed." };
  }
  
  // If auto-merge, it's succeeded
  if (mergeStrategy === "auto") {
    return { ...baseResult, status: "succeeded", summary: `Modified ${inspection.changedFiles.length} files.` };
  }
  
  // If interactive, it's blocked pending approval
  if (mergeStrategy === "interactive") {
    return { ...baseResult, status: "blocked", summary: `Pending review of ${inspection.changedFiles.length} files.` };
  }
  
  return baseResult;
};
```

### 4. Refactor `executeAttempt.ts`

This is the critical path. Replace the synchronous pipeline with the new decoupled flow.

```typescript
// src/host/executeAttempt.ts (conceptual outline)

export const executeAttempt = async (
  ctx: ExecuteAttemptContext,
  plan: DispatchPlan,
): Promise<AttemptExecutionResult> => {
  const taskId = generateSandboxTaskId(plan.spec.attemptId);
  const aboxArgs = buildAboxRunArgs(taskId, plan.profile, ctx.repoRoot);
  
  // 1. Execute the worker (waits for exit)
  const workerResult = await runInStreamLive(aboxArgs, plan.spec);
  
  let inspection: WorktreeInspection | null = null;
  let hostArtifacts: string[] = [];
  
  // 2. Discover and Inspect (if preserved)
  if (plan.profile.sandboxLifecycle === "preserved") {
    const snapshot = await discoverWorktree(ctx.repoRoot, taskId);
    if (snapshot) {
      inspection = await inspectWorktree(snapshot);
      hostArtifacts = await writeHostArtifacts(ctx.sessionDir, plan.spec.attemptId, inspection);
    }
  }
  
  // 3. Review the outcome based on the worktree state
  workerResult.artifacts.push(...hostArtifacts);
  const finalResult = reviewAttemptResult(workerResult, inspection, plan.profile.mergeStrategy);
  
  // 4. Handle Merge Strategy
  if (plan.profile.sandboxLifecycle === "preserved") {
    if (finalResult.status === "succeeded" && plan.profile.mergeStrategy === "auto") {
      await mergeSandbox("abox", ctx.repoRoot, taskId);
    } else if (finalResult.status === "failed" || plan.profile.mergeStrategy === "none") {
      await discardSandbox("abox", ctx.repoRoot, taskId);
    }
    // If blocked (interactive), do nothing — leave it preserved for Wave 5.
  }
  
  return finalResult;
};
```

## Test Strategy
- **Unit:** Test `worktreeInspector.ts` with mocked `execFile` outputs. Test `reviewer.ts` with various combinations of `hasChanges` and `mergeStrategy`.
- **Integration:** The live pipeline will now generate `patch.diff` files in the `.bakudo/sessions/` directory.

## Acceptance Criteria
- `pnpm test:unit` passes.
- A run with `mergeStrategy: "interactive"` leaves the worktree preserved and returns a `blocked` status.
- A run with `mergeStrategy: "auto"` merges the worktree and returns a `succeeded` status.

## Rollback
If the execution pipeline hangs or git discovery fails, ensure `discoverWorktree` catches errors and returns `null` safely. Revert `executeAttempt.ts` to the synchronous flow if necessary.
